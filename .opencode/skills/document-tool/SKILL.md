## What to do

- Search the web with tool name, summarize tool's purpose and add link to the website or git repo
- Load the current tool folder (it is always the top folder in repo, with the same name as the tool). Analyze folder contents, and summarize special configuration options or plugins.
- read @README.md, find there section with the name of the tool. If section does not exists, then create new, otherwise update current section

## Guidelines

- Never touch description or anything about other tools
- Only add essential information
- Tool section should be created as second tier header (##)
- Documentation should have subsections
